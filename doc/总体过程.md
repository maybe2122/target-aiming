
## 阶段一：固定目标锁定（入门验证）

**目标**：在 Isaac Lab 中验证整个 RL 训练闭环能跑通，策略能学会将云台对准静止目标。

**动作空间**（3维连续）：`[Δyaw, Δpitch, Δzoom]`，归一化到 `[-1, 1]`，分别映射到角速度和焦距变化速率。

**观测空间**：`(u, v)` 目标像素坐标 + `(yaw, pitch)` 当前关节角，共 4 维 MLP 输入。不上图像，先跑通控制逻辑。

**奖励函数**：
```
r = -normalized_pixel_error
  - 0.01 * ||action||²
  + 10.0 * (error < 0.05)   # 对准完成奖励
```

**验收标准**：平均 episode 奖励稳定上升，目标像素误差收敛至图像中心 5% 以内。达到后进入阶段二。

---

## 阶段二：联合控制扩展（车辆 + 云台）

**核心难点**：底盘运动会干扰云台视角，两者需要协同——车辆靠近目标同时云台持续对准。

**动作空间**（6维连续）：
```
[Δyaw, Δpitch, Δzoom,   # 云台
 vx, vy, ω]             # 底盘：前后、左右平移、原地旋转
```

**观测空间扩展**：在阶段一基础上加入底盘状态 `(vx, vy, ω, heading)`，总维度约 8 维（仍先用坐标输入）。验证联合控制后再替换为图像输入 + CNN Encoder。

**多目标奖励设计**（关键）：
```
r = w1 * (-pixel_error)           # 对准奖励（主项）
  + w2 * distance_reward           # 底盘靠近目标（如目标在视野边缘则驱动靠近）
  + w3 * (-chassis_jerk)           # 底盘运动平滑惩罚
  + w4 * visibility_bonus          # 目标保持在视野内的奖励
  + w5 * (-||action||²)            # 防抖

# 建议权重初值：w1=1.0, w2=0.3, w3=0.05, w4=0.5, w5=0.01
```

**课程学习（Curriculum）建议**：先固定底盘只训练云台（重用阶段一策略作为初始化），再解锁底盘自由度，避免探索空间爆炸。

**Isaac Lab 实现要点**：底盘用 `set_joint_velocity_target` 控制四轮差速或全向轮；用 `ArticulationData.body_pos_w` 获取底盘位置用于计算距离奖励。

## 阶段三：动目标跟踪（无人机飞行目标）

**核心难点**：目标轨迹不可预测，需要策略具备时序记忆能力，同时应对 sim-to-real 的外观差异。

**无人机目标设计**：
- 在 Isaac Lab 中用 `RigidObject` 或独立 `Articulation` 模拟无人机
- 飞行轨迹可以是：匀速直线 → 变速绕圈 → 随机 waypoint 追踪（逐步增加难度）
- 用 `EventTermCfg` 在每个 episode reset 时随机化初始位置、速度和飞行模式

**策略网络升级**：
- 加入时序模块（LSTM 或 Mamba），让策略能根据历史帧预测目标运动趋势
- 或使用帧差图（连续两帧相减）作为额外输入，编码运动信息

**Domain Randomization**（sim-to-real 关键）：
```python
# 视觉随机化（通过 Replicator API）
- 目标外观：随机颜色、纹理、大小
- 光照：随机方向光强度、环境光
- 背景：随机天空盒

# 物理随机化（EventTermCfg）
- 云台关节摩擦、阻尼
- 底盘轮胎摩擦系数
- 无人机飞行速度范围
```

**验收标准**：目标做随机飞行时，跟踪成功率（像素误差 < 10%）在测试环境中 > 80%。

## 最终系统总结

一个模型，双输入，6维输出：

| 输入 | 内容 |
|------|------|
| 图像 | `TiledCamera` RGB/Seg，84×84 |
| 坐标 | 目标像素位置 `(u, v)` + 云台角度 + 底盘速度 |

| 输出 | 控制量 |
|------|--------|
| 云台 | `Δyaw, Δpitch, Δzoom` |
| 底盘 | `vx（前后）, vy（左右）, ω（旋转）` |

三个阶段的逻辑是：先验证控制可行（一），再扩展执行器数量（二），最后增加目标复杂度（三）——每一步都在前一步收敛的基础上叠加一个变量，是降低失败风险的最稳健路径。
